# Cache Viewer Dashboard

A lightweight, zero-dependency web dashboard for visualizing and searching the massive JSON cache files generated by the CGalleryOrganizer stress tests.

## 1. Setup & Requirements

The dashboard is built with vanilla HTML5/JavaScript. Because it fetches the `perf_cache.json` file asynchronously, it **cannot** be opened directly by double-clicking the `index.html` file (due to browser CORS security policies).

You must serve it via a local web server.

### Option A: Using Python (Recommended)
Python is pre-installed on most systems.
1. Open your terminal in the **PROJECT ROOT**.
2. Start the server:
   ```bash
   python3 -m http.server 8000
   ```
3. Open: [http://localhost:8000/tools/cache_viewer/](http://localhost:8000/tools/cache_viewer/)

### Option B: Using Node.js (npx)
Run from the **PROJECT ROOT**:
```bash
npx serve .
```
And navigate to `/tools/cache_viewer/`.

## 2. Accessing the GUI

Once your server is running:
1. Open your web browser (Chrome, Firefox, Safari, etc.).
2. Navigate to: [http://localhost:8000](http://localhost:8000)
3. Click the **"Load Default"** button to instantly map the `build/perf_cache.json` generated by your latest stress test.
    - You can also manually upload any other `.json` cache file using the file picker.

## 3. Stopping and Cleaning Up

### To Stop the Dashboard:
Return to the terminal where the server is running and press:
`Ctrl + C`

### To Clean Performance Cache Data:
If you want to wipe the stress test telemetry and start fresh:
```bash
# Deletes ONLY the performance cache and binaries
make clean

# Deletes performance cache, binaries, AND the downloaded 1GB dataset
make clean-all
```

> [!NOTE]
> The dashboard is isolated. It does not touch your production `.cache/gallery_cache.json` unless you manually upload it via the UI for inspection.
